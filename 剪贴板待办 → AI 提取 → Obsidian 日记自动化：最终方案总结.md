# 剪贴板待办 → AI 提取 → Obsidian 日记自动化：最终方案总结

剪贴板待办 → AI 提取 → Obsidian 日记自动化：最终方案总结

经过多轮调试，成功实现了一个完全本地的自动化工作流：**复制任意文本（含待办事项）→ 运行 AppleScript → 通义千问自动提取任务和截止日期 → 写入 Obsidian 日记的“## 待办”标题下方**，并自动添加 🎯 标记、日期和来源应用标签。

### 一、最终成果：一个完整的 AppleScript 脚本

脚本实现了以下功能：

1. **读取剪贴板内容**，若为空则提示退出。
    
2. **获取当前前台应用名称**（如微信、QQ、浏览器等），作为待办来源。
    
3. **调用阿里云百炼 API（通义千问 qwen-plus 模型）**，使用系统指令强制提取 JSON 格式：`{"task": "任务内容", "due_date": "YYYY-MM-DD"}`。支持自然语言日期（如“明天”“本周五”）。
    
4. **解析 API 返回的 JSON**，提取任务和截止日期。
    
5. **将待办条目写入 Obsidian 日记**：
    
    - 日记路径：`/vault/04. 🔵 日记周记/01. 日记/YYYY-MM-DD.md`
        
    - 插入位置：`## 待办` 二级标题下方（若不存在需手动创建）
        
    - 格式：`- [ ] 🎯 任务内容 📅 截止日期 🛫 当天日期 #来源应用`
        
    - 通过 URL 编码处理中文和特殊字符，使用 `PATCH` 方法及必要请求头（`operation: append`、`target-type: heading`），并在内容前添加换行符确保新行插入。
        
6. **全程使用临时文件**（Python 生成 JSON 请求体、存储响应、存储待追加内容），避免 AppleScript 复杂的引号转义问题。
    
7. **显示通知**：成功或失败均有系统通知。
    

### 二、技术要点与踩坑记录

|环节|问题|解决方案|
|---|---|---|
|AppleScript 调用 curl 构建 JSON|引号嵌套、换行符转义极其繁琐|将 Python 代码写入临时脚本文件，用 `do shell script` 执行|
|阿里云 API 返回的 `false` 导致 Python 报错|JSON 中的 `false` 在 Python 中被当作变量|使用 `json.loads()` 正确处理，并捕获异常|
|Obsidian Local REST API 要求必须包含 `operation` 和 `target-type` 头|遗漏导致 `40053` 错误|添加 `-H "operation: append" -H "target-type: heading"`|
|插入到 `## 待办` 标题下但内容跑到了同一行|缺少换行或 `content-insertion-method=newline` 无效|在待办内容最前面加上 `\n`，并用 `printf` 写入临时文件|
|路径中的中文和 emoji 导致 404|未进行 URL 编码|使用 Python `urllib.parse.quote` 编码，保留 `/`|
|剪贴板文本含特殊字符导致 API 请求体损坏|直接拼接字符串|通过 Python `json.dumps` 转义后写入文件|
|获取前台应用名称|需要 System Events 权限|首次运行会弹出授权请求，允许即可|

### 四、最终脚本核心代码片段（写入部分）

```applescript
-- 构建待办行，开头加换行符，任务前加 🎯
set markdownLine to "\n- [ ] 🎯 " & taskItem
if dueDateItem ≠ "null" and dueDateItem ≠ "" then
    set markdownLine to markdownLine & " 📅 " & dueDateItem
end if
set markdownLine to markdownLine & " 🛫 " & today & " #" & frontApp
-- URL 编码路径
set encodedPath to do shell script "python3 -c 'import sys, urllib.parse; print(urllib.parse.quote(sys.argv[1], safe=\"/\"))' " & quoted form of rawPath
-- 写入临时文件（保留换行符）
set contentFile to tempDir & "/content.txt"
do shell script "printf '%s' " & quoted form of markdownLine & " > " & quoted form of contentFile
-- 发送 PATCH 请求
set writeURL to "http://localhost:27123" & encodedPath & "?heading=%E5%BE%85%E5%8A%9E&content-insertion-position=beginning"
set writeCmd to "curl -s -X PATCH \"" & writeURL & "\" -H \"Authorization: Bearer " & obsidianToken & "\" -H \"Content-Type: text/markdown\" -H \"operation: append\" -H \"target-type: heading\" --data-binary @" & quoted form of contentFile
do shell script writeCmd
```

### 🏆 方案一：使用 macOS 原生“自动操作”（推荐）

这是最直接、最稳定，且无需安装任何第三方软件的方法。

1. **保存你的脚本**  
    打开“脚本编辑器”，将你最终确认的 AppleScript 完整粘贴进去，并保存为一个应用程序（格式选择“应用程序”），例如命名为 `我的待办助手`。
    
2. **创建“快速操作”**
    
    - 打开 **“自动操作”** App（在“启动台”或“应用程序”文件夹中）。
        
    - 点击“新建文稿”，选择 **“快速操作”**。
        
    - 在工作流界面，按照下图所示设置：
        
        - **“工作流程收到”**：选择 **“没有输入”**。
            
        - **“位于”**：选择 **“任何应用程序”**。
            
    - 从左侧资源库的“实用工具”中，将 **“运行 AppleScript”** 拖拽到右侧工作区。
        
    - 将你保存好的脚本内容，**完整地复制并粘贴** 到弹出的代码框中，替换掉默认的 `(* Your script goes here *)`。

    按下 `⌘` + `S` 保存，给你的“快速操作”起一个清晰的名字，比如 `提取剪贴板待办`。
    
4. **设置快捷键**
    
    - 打开 **“系统设置”**，选择 **“键盘”**，然后点击 **“键盘快捷键”**。
        
    - 在左侧列表中选择 **“服务”**。
        
    - 在右侧列表的“通用”部分（通常在底部），找到你刚刚命名的 `提取剪贴板待办` 服务。
        
    - 双击其右侧的“无”，按下你希望设置的快捷键组合，例如 **`⌃` + `⌥` + `⌘` + `F`**，即可完成绑定[](https://cloud.tencent.cn/developer/article/1908780)[](https://codeberg.org/ju4b/macos-services)。

- **为什么没有集中的设置界面？**  
    FastScripts 的设计哲学是“在脚本所在处设置快捷键”，即直接在菜单项上通过 `⌘+点击` 来绑定。这确实不够直观，但熟悉后很方便。

- **如果 `⌘+点击` 没有反应？**  
    请检查：
    
    - 确保你**先按住 `⌘` 键，再点击**（不要先点击后按）。
        
    - 确保 FastScripts 在“辅助功能”中有权限（系统设置 → 隐私与安全性 → 辅助功能 → 添加 FastScripts）。
        
    - 尝试重启 FastScripts（菜单栏图标 → Quit，然后重新打开）。

## ⚙️ 首次运行时的权限设置

由于你的脚本需要访问**剪贴板**、**系统事件**（获取前台应用）以及**发送网络请求**，首次双击运行应用程序时，系统会弹出权限提示：

    - 无需额外设置，但首次使用时系统可能会提示，允许即可。
        

> **建议**：在设置好权限之前，可以先从脚本编辑器运行一次，一次性批准所有请求。然后导出的应用程序会继承这些权限（可能需要重新授权一次）。
